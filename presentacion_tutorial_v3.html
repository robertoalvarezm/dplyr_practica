¬øPor qu√© dplyr?

El problema con R base

R base ‚Äî funciona, pero se lee de adentro hacia afuera:

head(
  vinos[vinos$quality >= 8,
        c("tipo", "quality", "alcohol")],
  5
)

dplyr ‚Äî se lee como una receta:

vinos |>
  filter(quality >= 8) |>
  select(tipo, quality, alcohol) |>
  head(5)

dplyr no reemplaza a R base ‚Äî lo hace m√°s legible.
El vocabulario completo son 6 verbos.

Los 6 verbos

Verbo ¬øQu√© hace?

arrange() Ordena filas
mutate() Crea o modifica columnas
summarise() Resume la tabla a estad√≠sticos

Verbo 1: filter()

Quedarte solo con ciertas filas

filter() ‚Äî La idea

R base:

vinos[vinos$quality == 9, ]

dplyr:

filter(vinos, quality == 9)

No necesitas vinos$quality ‚Äî dplyr ya sabe a qu√© tabla te refieres.

AND ‚Üí coma o &:

filter(vinos,
       tipo == "tinto",
       quality >= 8)

filter(salud,
       SmokingStatus == "Fuma",
       BMI > 25)

filter() ‚Äî OR, %in%, rangos

OR ‚Üí |:

filter(vinos,
       quality == 9 | alcohol > 14)

Varios valores ‚Üí %in%:

filter(vinos, quality %in% c(8, 9))

filter(salud,
       ResidenceType %in%
         c("Urbano", "Suburbano"))

Rango ‚Üí between():

filter(vinos, between(pH, 3.0, 3.1))

filter(salud, between(Age, 25, 55))

Incluir NA expl√≠citamente:

filter(salud,
       is.na(Age) | Age < 30)

filter() ‚Äî Error frecuente y ejercicio

‚ö†Ô∏è = vs ==

filter(vinos, quality = 8)    # ‚ùå  = es asignaci√≥n
filter(vinos, quality == 8)   # ‚úÖ  == es comparaci√≥n

Verbo 4: mutate()

Crear o modificar columnas

mutate() ‚Äî ifelse() y case_when()

üç∑ ifelse ‚Äî clasificaci√≥n binaria:

vinos <- vinos |>
  mutate(
    calidad_alta = ifelse(
      quality >= 7, "S√≠", "No"
    )
  )

case_when ‚Äî m√∫ltiples categor√≠as:

vinos <- vinos |>
  mutate(
    categoria = case_when(
      quality <= 4 ~ "Baja",
      quality <= 6 ~ "Media",
      quality <= 8 ~ "Alta",
      TRUE         ~ "Excepcional"
    )
  )

üè• salud ‚Äî NA como primera condici√≥n:

salud <- salud |>
  mutate(
    categoria_imc = case_when(
      is.na(BMI)  ~ "Sin dato", # ‚Üê primero
      BMI < 18.5  ~ "Bajo peso",
      BMI < 25    ~ "Normal",
      BMI < 30    ~ "Sobrepeso",
      TRUE        ~ "Obesidad"
    )
  )

Advertencia

Sin is.na() primero, los NA caen en TRUE y se clasifican incorrectamente.

Funciones √∫tiles en summarise()

Funci√≥n ¬øQu√© hace?

Joins

Combinar dos tablas por una llave en com√∫n

Los joins principales

Tabla A:  id1  id2  id3  id4
Tabla B:       id2  id3  id4  id5

inner_join() ‚Üí {id2, id3, id4}
left_join()  ‚Üí {id1, id2, id3, id4}  ‚Üê el m√°s usado
full_join()  ‚Üí {id1, id2, id3, id4, id5}
anti_join()  ‚Üí {id1}  (los de A sin coincidencia en B)
inner_join(clientes, pedidos,
           by = "id")

left_join(clientes, pedidos,
          by = "id")
full_join(clientes, pedidos,
          by = "id")

anti_join(clientes, pedidos,
          by = "id")
